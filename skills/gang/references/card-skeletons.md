# Gang GitHub Card Skeletons

Each skeleton defines the required structure for a specific card type.
The `/gang push` workflow reads this file, selects the matching skeleton,
and fills every `{placeholder}` from the evaluation data before sending
the card to GitHub Projects.

Fields marked `> 📌 Required` must be filled from evaluation data.
Fields marked `> 📝 Review` are pre-filled but should be reviewed.
Fields marked `> ✏️ Manual` are left as placeholders for the team to fill.

---

## Card Types

| Type ID       | Label           | When to use |
|---------------|-----------------|-------------|
| `feature`     | 🚀 Feature       | New functionality evaluated and approved |
| `enhancement` | ✨ Enhancement   | Improvement to an existing feature/system |
| `initiative`  | 🏢 Initiative    | Strategic business-level decision |
| `change`      | 🔄 Change Request | Modification to existing functionality or process |
| `spike`       | 🔬 Research Spike | Time-boxed investigation before committing |

---

## Priority Derivation

Derive `{priority}` from the Gang weighted average score:

| Weighted Average | Priority |
|-----------------|----------|
| 8.0 – 10.0 | 🔴 High |
| 5.0 – 7.9  | 🟡 Medium |
| 0.0 – 4.9  | 🟢 Low |
